02

Widget anatomy

The inline widget has four layers, top to bottom.

contextus powered by contextus
Hi! How can I help you today?
I need help with tax filing
LAYERDESCRIPTION
HeaderLogo mark (18px) + "contextus" + "powered by contextus" right-aligned. 0.5px bottom border. Customizable: business owner can replace "contextus" with their own name.
Message areaAgent messages: logo avatar (24px) + left-aligned plain text. Visitor messages: right-aligned dark bubbles. Auto-grows. Scrolls at max-height.
Input area#f0f0f0 background, 16px radius, 15px font. Send button pinned right inside the input wrapper. Always visible — never scrolls out of view.
Pills (idle only)Below input. 11px text, 14px radius, 0.5px border. Disappear after first message. Optional — omit if knowledge base is too thin.

04

Widget states

The inline widget has 11 distinct states. Each state defines what the visitor sees, how the input behaves, and what the system does behind the scenes.

STATE 1
Idle

First load. Input empty, send button gray. Pills suggest starting points.

contextuspowered by contextus
How can you help?
Services & pricing
Talk to a human
Send button: EMPTY (gray). Pills: Auto-generated from knowledge base top topics, or defined by owner during onboarding. No pills if KB is too thin — no pills is better than bad pills. Header: Customizable name. Height: Compact.
STATE 2
Visitor typing

Text in input. Send button turns black.

contextuspowered by contextus
How can you help?
Services & pricing
Talk to a human
Send button: ACTIVE (black). Click or Enter sends. On send: Visitor bubble appears, input clears, pills disappear, typing indicator shows, input disables. On clear: Button returns to EMPTY.
STATE 3
Agent thinking

Message sent. Agent processing. Input disabled, send button gray + not-allowed.

contextuspowered by contextus
Saya perlu bantuan lapor SPT untuk PT baru
Send button: DISABLED (gray, not-allowed). Input disabled. Placeholder: "Waiting for response..." On response: Input re-enables, placeholder resets, button goes EMPTY. This is the ONLY state where send button is DISABLED.
STATE 4
Active conversation

Multi-turn dialogue. Input ready, send button empty.

contextuspowered by contextus
Great, F&B has specific tax considerations. How many employees?
About 15 orang
With 15 employees you'll need PPh 21 withholding. Would you like our specialist to reach out?
Send button: EMPTY → ACTIVE on keystroke. Agent: Max one question per turn. Tags silently: industry, size, needs, urgency. Height: Grows with messages. Scroll at max-height.
STATE 5
Long conversation (scroll)

Content exceeds max-height. Scroll appears. Input pinned at bottom.

Max-height: 400px desktop, 50vh mobile. Scroll: Auto-scrolls to latest. If visitor scrolled up, show "new message" indicator. Scrollbar: Thin 3px, minimal chrome. Input: Pinned at bottom, always visible. Same EMPTY/ACTIVE/DISABLED rules.
STATE 6
Knowledge boundary hit

Agent can't answer. Redirects to human. The conversion moment.

contextuspowered by contextus
What is the PPh 21 rate for salary of 15 million?
That's a great question — the exact rate depends on factors specific to your situation. I want to make sure you get the accurate answer.
Redirect: "Would you like our tax specialist to walk you through this? WhatsApp or email?"
Trigger: RAG low confidence, or guardrail topic match from onboarding. Agent: Does NOT hallucinate. Acknowledges, explains, pivots to contact capture. Lead brief: Flagged as "unanswered question" — the follow-up hook.
STATE 7
Contact captured

Visitor shared contact info voluntarily. Agent confirms. Conversation continues.

Trigger: Contact pattern detected (phone, email, WhatsApp mention). Agent: Confirms naturally ("I've noted that"). Doesn't end conversation. System: Lead brief generation starts immediately. Business owner notified. Visual: No visible indicator to visitor. Debug tag "CONTACT CAPTURED" is internal only.
STATE 8
Visitor idle

60+ seconds silence mid-conversation. One gentle nudge.

contextuspowered by contextus
How many employees do you have?
Still there? No rush — I'm here whenever you're ready.
Nudge: Muted italic text. NOT a chat bubble. Max ONE per conversation. After 5 min idle: Conversation ends silently. Lead brief generated.
STATE 9
Error

Network failure, LLM timeout, or rate limit.

contextuspowered by contextus
Can you help with annual tax filing?
Something went wrong. Please try again in a moment.
Auto-retry: One silent retry after 2s before showing error. Display: Inline, muted red (#FCEBEB bg, #A32D2D text). Disappears on next successful send. Rate limit: "Please wait a moment" — never reveal it's a rate limit. Input: Active for retry.
STATE 10
Returning visitor

Previous conversation at reduced opacity. Can continue or start new.

Trigger: Return within session (tab lifespan or 30 min). Previous messages: 55% opacity + "Previous conversation" label with clock icon. Input placeholder: "Continue or start new..." Session expiry: After 30 min or tab close, full reset to idle. No localStorage: Memory only. No cookies.
STATE 11
Conversation complete

Natural end. Lead brief generated and sent to business owner.

contextuspowered by contextus
You're welcome! Have a great day.
Conversation ended — lead brief sent
Trigger: Completion signal ("thanks", "that's all") or 5 min idle after contact captured. Agent: Warm sign-off. No rating/feedback request. System: Final lead brief generated. Full conversation + extracted data + score + suggested approach. Soft-reset: After 30s, placeholder changes to "Start a new conversation..."

06

Embed methods

Two tiers, from easiest to most powerful. Both render the same widget. Offer both to customers — let them choose based on their technical comfort.

Tier 1: iframe URL (easiest — MVP default)

Works on every website builder. The business owner pastes a URL. That's it.

<iframe src="https://contextus.ai/embed/YOUR_SITE_ID?transparent=1&dynamicHeight=1" width="100%" height="400" frameborder="0" style="border: none; min-width: 320px;" ></iframe>
PARAMETERDEFAULTDESCRIPTION
transparent0Transparent background (blends with host page)
dynamicHeight0Auto-resize iframe via postMessage to parent
langauto"en", "id", or "auto" (detect from first message)
greeting"Ask us anything..."Custom input placeholder text
name"contextus"Custom header name

The server renders the full widget at /embed/:site_id as a standalone HTML page. This is the same widget component used in Tier 2 — just wrapped in a minimal page shell.

Tier 2: div + script (inline — beautiful)

For customers who can paste HTML. The script injects the widget directly into the page — no iframe, fully native.

<div data-contextus="YOUR_SITE_ID"></div> <script src="https://cdn.contextus.ai/embed.js" async></script>
ATTRIBUTEREQUIREDDESCRIPTION
data-contextusYesSite identifier. The script scans for all elements with this attribute.
data-nameNoCustom header name (default: "contextus")
data-greetingNoCustom input placeholder
data-langNo"en", "id", or "auto" (default: auto-detect)

Tier 3: platform plugins (future — not MVP)

WordPress plugin, Wix app, Shopify app — only build when paying customers demand them. For Webflow, Squarespace, and Framer, documentation guides are sufficient since they support raw HTML natively.

Implementation details are subject to change. This spec defines the design contract — visual appearance and behavior — not the final technical API.
